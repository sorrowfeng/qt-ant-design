Guide

快速开始

使用 CMake 将 qt-ant-design 接入 Qt Widgets 项目，并完成主题、资源、字体和高 DPI 初始化。

环境要求 最短路径 子目录接入 安装包接入 启动初始化 运行示例

环境要求

  • Qt 5.15.2 或 Qt 6.x，至少需要 Core、Widgets、Svg 模块。
  • CMake 3.16+，C++17 编译器。
  • Windows 下推荐启用高 DPI 感知，项目已提供统一启动入口。

最短路径

首次 clone 后可以先跑通 example，再决定是随业务工程一起构建，还是安装后通过 CMake package 消费。

1. 配置 Qt 套件

把 CMAKE_PREFIX_PATH 指向当前 Qt5 或 Qt6 安装目录。

2. 构建 example

先验证示例窗口、图标、弹层和高 DPI 表现是否符合预期。

3. 接入业务工程

使用 AntDesign::initialize(&app) 完成资源、字体和主题初始化。

cmake -S . -B build -DCMAKE_PREFIX_PATH=/path/to/Qt -DBUILD_SHARED_LIBS=OFF
cmake --build build --config Debug --target qt-ant-design-example

作为子目录接入

适合和业务工程一起构建，库类型通过标准 BUILD_SHARED_LIBS 控制。

add_subdirectory(qt-ant-design)

find_package(QT NAMES Qt6 Qt5 REQUIRED COMPONENTS Core Widgets Svg)
find_package(Qt${QT_VERSION_MAJOR} REQUIRED COMPONENTS Core Widgets Svg)

target_link_libraries(my-qt-app PRIVATE
    Qt${QT_VERSION_MAJOR}::Core
    Qt${QT_VERSION_MAJOR}::Widgets
    qt-ant-design
)

安装包接入

cmake -S . -B build -DCMAKE_PREFIX_PATH=/path/to/Qt
cmake --build build --config Release --target install

消费项目中使用 CMake package：

find_package(qt-ant-design CONFIG REQUIRED)
target_link_libraries(my-qt-app PRIVATE qt-ant-design::qt-ant-design)

启动初始化

推荐在创建 QApplication 前配置高 DPI，创建应用后初始化资源、字体和主题。若你希望统一使用 initialize()，也可以在 QApplication 前调用一次无参数 AntDesign::initialize()，它会完成 High DPI 预配置并返回；创建 app 后仍需再次调用 AntDesign::initialize(&app)。

int main(int argc, char* argv[])
{
    AntDesign::configureHighDpi();
    QApplication app(argc, argv);
    AntDesign::initialize(&app);

    auto* button = new AntButton;
    button->setText(QStringLiteral("Primary"));
    button->setButtonType(Ant::ButtonType::Primary);
    button->show();

    return app.exec();
}

运行示例

cmake -S . -B build -DCMAKE_PREFIX_PATH=/path/to/Qt
cmake --build build --config Debug --target qt-ant-design-example
示例程序使用 AntWindow、AntNav 和 AntScrollArea 组织所有组件页面，是检查主题、滚动、缩放比例和 Qt5/Qt6 视觉一致性的主入口。
