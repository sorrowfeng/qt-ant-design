# qt-ant-design

`qt-ant-design` is a Qt6 Widgets component library that ports the Ant Design design system to native C++ widgets with hand-painted `QPainter` rendering.

It focuses on dynamic light/dark theme switching, Design Token driven styling, and faithful Ant Design interactions for desktop applications. The rendering architecture is inspired by [ElaWidgetTools](https://github.com/Liniyous/ElaWidgetTools), while visual behavior and component specifications follow [Ant Design](https://github.com/ant-design/ant-design).

## Features

- Qt6 Widgets based, lightweight, static-library friendly, and easy to embed in existing desktop projects.
- Built-in Design Token system with real-time light/dark theme switching.
- Hand-painted components with `QPainter`, avoiding QSS as the primary rendering path.
- 22 migrated core components, with more Ant Design components being added continuously.
- Complete example application showing component variants, interaction states, and theme switching.
- Clear project structure for extending themes, palettes, and widgets.

## Installation And Integration

### Requirements

- Qt 6.5+
- CMake 3.16+
- C++17 compiler

### Method 1: Add As A CMake Submodule

```bash
git submodule add https://github.com/sorrowfeng/qt-ant-design.git third_party/qt-ant-design
git submodule update --init --recursive
```

```cmake
cmake_minimum_required(VERSION 3.16)
project(my-qt-app LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

find_package(Qt6 REQUIRED COMPONENTS Core Widgets)

add_subdirectory(third_party/qt-ant-design)

add_executable(my-qt-app main.cpp)
target_link_libraries(my-qt-app PRIVATE Qt6::Core Qt6::Widgets qt-ant-design)
```

### Method 2: Build And Link The Static Library

```bash
cmake -S . -B build -DCMAKE_INSTALL_PREFIX=/path/to/install
cmake --build build --config Release
cmake --install build --config Release
```

Then add the installed include directory and link the generated static library from `install/lib`.

```cmake
find_package(Qt6 REQUIRED COMPONENTS Core Widgets)

add_executable(my-qt-app main.cpp)
target_include_directories(my-qt-app PRIVATE /path/to/install/include/qt-ant-design)
target_link_libraries(my-qt-app PRIVATE
    Qt6::Core
    Qt6::Widgets
    /path/to/install/lib/qt-ant-design.lib
)
```

On Linux or macOS, replace the library suffix/path with the platform-specific static library generated by CMake.

## Quick Start

```bash
git clone https://github.com/sorrowfeng/qt-ant-design.git
cd qt-ant-design
mkdir build && cd build
cmake .. -DCMAKE_PREFIX_PATH=/path/to/Qt6
cmake --build .
./examples/qt-ant-design-example
```

On Windows with the installed example:

```powershell
cmake -S . -B build -DCMAKE_INSTALL_PREFIX=D:/Project/GitProject/qt-ant-design/install
cmake --build build --config Debug
cmake --install build --config Debug
.\install\bin\qt-ant-design-example.exe
```

### Minimal Window With AntButton

```cpp
#include <QApplication>
#include <QVBoxLayout>
#include <QWidget>

#include "widgets/AntButton.h"

int main(int argc, char* argv[])
{
    QApplication app(argc, argv);

    QWidget window;
    auto* layout = new QVBoxLayout(&window);

    auto* button = new AntButton("Primary Button");
    button->setButtonType(Ant::ButtonType::Primary);
    layout->addWidget(button);

    window.resize(360, 200);
    window.show();

    return app.exec();
}
```

## Usage Examples

### AntButton

```cpp
#include "widgets/AntButton.h"

auto* primary = new AntButton("Save");
primary->setButtonType(Ant::ButtonType::Primary);
primary->setButtonSize(Ant::ButtonSize::Middle);

auto* danger = new AntButton("Delete");
danger->setDanger(true);
```

### AntInput

```cpp
#include "widgets/AntInput.h"

auto* input = new AntInput();
input->setPlaceholderText("Please enter a name");
input->setAllowClear(true);
input->setInputSize(Ant::InputSize::Large);
```

### AntCard

```cpp
#include <QLabel>

#include "widgets/AntCard.h"

auto* card = new AntCard("User Profile");
card->setExtra("More");
card->setHoverable(true);
card->bodyLayout()->addWidget(new QLabel("Card content"));
```

### Theme Switching

```cpp
#include "core/AntTheme.h"

AntTheme::instance()->setThemeMode(Ant::ThemeMode::Dark);
AntTheme::instance()->setThemeMode(Ant::ThemeMode::Default);
AntTheme::instance()->toggleThemeMode();
```

Widgets connect to `AntTheme::themeChanged` or `AntTheme::themeModeChanged` and repaint automatically when the theme changes.

## Development And Contribution

This project uses `AGENT.md` as the AI collaboration and development convention document. It records project positioning, migrated components, pending components, implementation rules, example coverage, and install/run notes.

When adding a new component:

1. Read the matching Ant Design component docs and style source under `submodules/ant-design/components/<component>/`.
2. Study a similar ElaWidgetTools widget for `paintEvent`, animation, event handling, and class structure.
3. Add `src/widgets/Ant<Name>.h` and `src/widgets/Ant<Name>.cpp`.
4. Add shared enums to `src/core/AntTypes.h` when needed.
5. Use `AntTheme::instance()` for colors, fonts, spacing, radius, and state colors.
6. Render the component with `QPainter`; do not rely on QSS for the primary widget appearance.
7. Add an example page or extend `examples/ExampleWindow.cpp`.
8. Update `AGENT.md` and keep the migrated/pending component list accurate.

Issues and pull requests are welcome. Bug reports with screenshots, platform details, Qt version, and reproduction steps are especially helpful.
